What is the Command Line?

The command line is a text-based interface that allows you to interact with your computer's files and programs. It is powerful and often used by developers to perform various tasks quickly. One popular command line tool for working with Git is Git Bash.

Basic Command Line Commands

  • cd - Change directory. Example: cd Documents
  • ls - List files and folders in the current directory. Example: ls
  • mkdir - Create a new directory. Example: mkdir my-folder
  • touch - Create a new file. Example: touch file.txt
  • rm - Delete a file. Example: rm file.txt
  • clear - Clear the terminal screen.

Using Git Bash for Git

Git Bash is a command line tool that lets you use Git commands. Here are some common Git commands to get started:

  • git init - Initialize a new Git repository in your current directory.
  • git clone [URL] - Clone a repository from GitHub to your local machine.
  • git add . - Stage all changes in the current directory.
  • git commit -m "Your message" - Commit changes with a message describing the update.
  • git push - Push your commits to a remote repository like GitHub.
  • git status - Check the status of your repository, including any changes that need to be committed.

Example: Cloning a Repository


# Open Git Bash and run:
git clone https://github.com/your-username/your-repository.git

# Navigate into the cloned repository:
cd your-repository

# Make changes to a file and then stage the changes:
git add .

# Commit the changes:
git commit -m "Added new feature"

# Push changes to GitHub:
git push
